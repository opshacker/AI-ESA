## 核心问题域

### AI 系统面临的独特攻击面

AI 系统引入了传统软件不具备的攻击向量。这些风险需要在架构设计阶段予以识别，并在运行时持续监控。下表归纳了主要攻击类型及其控制要点。

| 攻击类型   | 机制描述                         | 典型场景               | 控制要点           |
| ---------- | -------------------------------- | ---------------------- | ------------------ |
| 对抗样本   | 精心构造的输入绕过模型检测       | 恶意软件检测、图像识别 | 对抗训练、输入验证 |
| 模型投毒   | 污染训练数据或模型参数           | 推荐系统、风控模型     | 数据溯源、异常检测 |
| 提示词注入 | LLM 应用的元指令攻击             | 对话机器人、代码生成   | 输入过滤、权限隔离 |
| 模型窃取   | 通过 API 推理反推模型逻辑        | MLaaS 服务             | 速率限制、输出扰动 |
| 隐私泄露   | 训练数据敏感信息通过输出泄露     | 医疗 AI、金融模型      | 差分隐私、输出脱敏 |
| 供应链风险 | 第三方模型、预训练权重的安全隐患 | 模型市场、开源框架     | 签名验证、来源审计 |

上述攻击类型的共同特征是：传统安全工具（防火墙、WAF、DLP）对其检测效果有限，需要针对性的 AI 安全控制措施。

### 合规压力：全球 AI 法规演进

企业在部署 AI 系统时，需评估并满足相关法规要求。不同法规的适用范围、违规后果、实施时间线存在差异，需根据业务所在地和行业特性进行合规规划。

下表列举主要 AI 相关法规标准的核心要求。选择这些维度的依据是：生效时间决定合规紧迫性，核心要求决定实施范围，适用场景决定是否需要纳入合规规划。

| 法规/标准                  | 生效时间 | 核心要求                     | 适用场景                 |
| -------------------------- | -------- | ---------------------------- | ------------------------ |
| EU AI Act                  | 2024.08  | 风险分级、高风险 AI 强制要求 | 在欧盟提供 AI 服务       |
| NIST AI RMF 1.0            | 2023.01  | 风险管理框架四大职能         | 美国联邦机构及相关供应商 |
| ISO/IEC 42001:2023         | 2023.12  | AI 管理体系认证              | 需第三方认证的企业       |
| 中国生成式 AI 管理暂行办法 | 2023.08  | 算法备案、内容安全           | 境内生成式 AI 服务       |

适用边界：上述法规适用于对外提供 AI 服务或在特定监管辖区运营的企业；内部研发用途、概念验证阶段可能有豁免条款，需具体分析。

关键约束：合规实施涉及组织调整（如设立 AI 治理委员会）、技术改造（如可解释性机制）、流程变更（如风险评估流程），需预留充足资源。

### AI 安全事件的业务影响

| 影响类型     | 表现形式           | 典型场景                   | 影响程度 |
| ------------ | ------------------ | -------------------------- | -------- |
| 信任损失     | 用户流失、品牌受损 | AI 误判、偏见决策曝光      | 长期     |
| 监管处罚     | 罚款、业务限制     | 违反 EU AI Act（第 99 条） | 高额     |
| 法律责任     | 诉讼、赔偿         | AI 决策导致歧视或伤害      | 不可预测 |
| 运营中断     | 业务停摆           | 反欺诈模型失效             | 直接     |
| 知识产权风险 | 竞争优势丧失       | 模型窃取                   | 战略性   |

## AI 全生命周期安全控制框架

本章构建覆盖 AI 全生命周期的安全控制体系。每个阶段的风险和控制措施如下：

```
AI 生命周期安全控制
│
├─ 数据采集与准备
│   ├─ 风险：数据投毒、隐私泄露、偏见引入
│   └─ 控制：数据验证、溯源、脱敏、质量检查
│
├─ 模型开发与训练
│   ├─ 风险：模型投毒、后门植入、算法偏见
│   └─ 控制：代码审查、环境隔离、对抗测试、偏见评估
│
├─ 模型评估与验证
│   ├─ 风险：评估数据污染、对抗样本绕过
│   └─ 控制：独立评估、红队测试、A/B 测试
│
├─ 模型部署与集成
│   ├─ 风险：供应链攻击、配置错误、权限滥用
│   └─ 控制：签名验证、最小权限、监控告警
│
├─ 模型推理与服务
│   ├─ 风险：输入攻击、输出操纵、DoS、模型窃取
│   └─ 控制：输入验证、输出过滤、速率限制、水印
│
└─ 模型监控与退役
    ├─ 风险：模型漂移、性能退化、持续攻击
    └─ 控制：性能监控、异常检测、安全更新、安全销毁
```

**验证方法**

| 验证方法 | 验证时机     | 验证目标       | 判定标准           |
| -------- | ------------ | -------------- | ------------------ |
| 设计评审 | 架构设计阶段 | 控制措施完整性 | 覆盖全生命周期风险 |
| 红队测试 | 部署前/定期  | 防护能力有效性 | 攻击成功率低于阈值 |
| 持续监控 | 运行时       | 控制持续有效   | 指标无异常偏离     |

**运行指标**

| 指标类别 | 具体指标              | 告警阈值表达    | 触发动作       |
| -------- | --------------------- | --------------- | -------------- |
| 模型性能 | 准确率/召回率/F1 偏离 | 偏离基线 > 阈值 | 模型复核       |
| 输入异常 | 疑似注入攻击请求占比  | 占比突增        | 规则加固       |
| 输出审计 | 触发敏感信息过滤次数  | 超出预期范围    | 过滤规则审查   |
| 漂移检测 | 特征分布偏移程度      | 偏移超阈值      | 模型重训练评估 |
